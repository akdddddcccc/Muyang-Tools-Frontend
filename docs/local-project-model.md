# 本地项目与融合画板模型

第一阶段不依赖账号、数据库或模型 Provider。当前浏览器使用 IndexedDB 保存一个本地项目：

- 数据库：`muyang-live-sticker`
- 存储：`projects/current`
- 图片：以 `Blob` 和素材元数据一起保存，页面恢复后重新生成仅供预览的 object URL。
- 融合文档：保存 9:16 比例、图层资产引用、选中图层、位置、尺寸、透明度、可见性和层级。

上传新底图、上贴、下贴、侧贴或文字图层时，最新素材会替换融合画板内同类型图层的资产引用，历史素材仍留在项目资产中，可用于后续版本/回退能力。

## 遮罩预留

每个融合图层都已有以下未开放字段：

- `mask.mode`：默认羽化或手绘遮罩。
- `mask.feather`：羽化强度。
- `mask.fadePath`：上贴/下贴的手绘渐隐边界曲线，支持 Shift 水平直线。上贴保留曲线以上，下贴保留曲线以下，边界在预览与导出中使用同一套羽化计算。
- `mask.edgeTexture`：`none`、`flame`、`cloud`。这是上贴/下贴透明边缘纹理的后期高级能力，第一期不开放。

预览画板和未来导出都必须读取同一份融合文档，避免出现预览羽化与导出不一致的问题。

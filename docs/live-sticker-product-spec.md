# 直播贴片工作台：第一阶段产品规格

## 工作区原则

背景生成、文字图层、效果融合与导出资产是四个可独立使用的工作区，不是严格锁死的步骤向导。用户可在任意工作区上传素材，也可复用同一项目中的资产。

## 背景生成

- 上贴、下贴、侧贴是独立生成任务，支持独立失败、重生与版本保留。
- 上贴完成后，文字图层可立刻使用；下贴、侧贴与文字层允许并行生成。

## 文字图层

- 默认以项目上贴为色彩、材质和装饰参考。
- 用户可上传自定义色彩纹理参考，覆盖默认上贴；因此没有背景任务时仍可独立使用。
- 默认只使用一个去色后的字体参考；它仅影响字形、笔画和局部纹理。
- 第二张字体参考为后期高级能力，不是第一阶段默认请求的一部分。
- 内置三组可持久化的生图字体参考：优雅宋体、表现书法、圆润可爱；它们只控制字形、笔画节奏和局部纹理，不覆盖上贴或用户色彩纹理参考。
- 任何进入融合画板的透明文字 PNG 都必须先按有效 alpha 像素边界预剪裁，并保留约 2.5% 的安全边距，用于保留描边、阴影和发光。画板按裁剪后的真实内容比例缩放，不能把大面积透明留白当作文字尺寸。

## 效果融合

- 可独立上传直播间无贴片底图、上贴、下贴、侧贴与文字图层。
- 原始贴片可以尚未做透明渐变；融合器后续提供默认羽化、手绘修正、Shift 水平直线、拖拽缩放与撤销恢复。
- 预览与导出使用同一份图层与遮罩数据。

## 导出

- 背景资产优先 JPEG；文字层、遮罩和合成结果使用 PNG。
- 用户选择已就绪资产后导出 ZIP。
- 项目配置 JSON 是后期高级能力；第一期不实现，界面保持禁用状态。

## 安全与连接

浏览器只能读取 `VITE_CORE_API_BASE_URL`，模型 Provider key 仅存在于 Core 服务端环境变量。Electron 与服务器侧可继续使用不暴露给浏览器的 `CORE_API_BASE_URL`。

## 当前基础状态

- 浏览器已具备本地项目资产模型：上传图片后会按素材类型加入当前项目，并可跨四个工作区复用或移除。
- 项目资产的图片 Blob、图层、选中状态、位置、尺寸、透明度与遮罩预留字段保存到浏览器 IndexedDB；刷新页面后可继续编辑。跨设备同步、服务端数据和项目配置 JSON 导出仍是后续切片。
- Frontend 使用 `GET /health` 探测 `live-sticker-api`；健康接口只报告服务基础状态，不读取 Provider key，也不调用模型。

## 本地融合画板

- 底图、上贴、下贴、侧贴和文字图层的最新上传素材会自动进入画板同类图层。
- 画板支持选择图层、拖拽移动、右下角缩放控制点、横向位置、纵向位置、宽度、高度、透明度与显示状态；画板编辑支持最多 50 步撤销/恢复。
- 默认羽化只作用于上贴和下贴：上贴在默认边界线以下过渡至透明，下贴在线以上过渡至透明。手绘渐隐会在上贴或下贴区域记录一条边界曲线，不是橡皮擦；上贴保留曲线以上，下贴保留曲线以下。按住 Shift 可画水平直线，边界保存到 `mask.fadePath`。
- 透明边缘纹理字段已预留，但火焰/云层等边缘样式仍为后期高级能力。
